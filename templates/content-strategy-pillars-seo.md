# Content: Strategy Pillars + SEO System

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Create a content strategy that compounds: clear pillars tied to ICP pains, a lightweight SEO plan, and a distribution loop so content isn’t published and forgotten.

## WHEN TO USE
Kick this off when:
- Content feels random or reactive.
- SEO is “we should do SEO” without a plan.
- You need content to reliably feed pipeline/activation.
- A new quarter starts and you need a content program.

This is upstream of the editorial calendar.

## INPUTS (MINIMUM)
- Target ICP(s)
- Primary business objective (pipeline, activation, retention)
- Core product promise and differentiators

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Existing content performance (top posts, top keywords)
- Competitor content examples
- Sales/support objections and FAQs

## PROCESS
1. **Define 3–5 pillars** anchored to ICP pains and buying triggers.
2. **Map each pillar to formats**: longform, shortform, video, webinars, case studies.
3. **SEO wedge**: choose 10–30 high-intent topics to start (not 500).
4. **Create the ‘proof library’ plan**: what case studies and data assets you need.
5. **Distribution loop**: how each piece will be repurposed into social/email/sales assets.
6. **Set cadence and quality bar**: what you can reliably ship.
7. **Define success metrics**: leading (engagement) + lagging (pipeline/activation).

## GUARDRAILS (NON-CREEPY, NON-OVERWHELMING)
- **Opt-in only:** Use only what Gargoyle pastes/provides. Never imply you “saw” private data.
- **Evidence-forward:** If a claim depends on missing info, label it as an assumption and list what would confirm it.
- **Decision-first:** Don’t dump frameworks. Produce an artifact Gargoyle can use immediately.
- **Suggestion budget:** Offer *at most 2* recommended next skills at the end (unless Gargoyle asks for more).
- **No creep:** No sentiment guesses about people. No “what they really mean.” Stick to facts, incentives, commitments, and risks.
- **Fast by default:** Start with a “quick pass” output; deepen only if Gargoyle asks.
- **Avoid content vanity:** If a pillar won’t drive action or trust with your ICP, cut it.
