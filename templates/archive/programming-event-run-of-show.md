# Programming: Event Run-of-Show

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Turn an event/webinar into a tightly run program: agenda, speakers, transitions, tech checks, promotion, and follow-up — so it drives outcomes instead of being “a nice event.”

## WHEN TO USE
Kick this off when:
- You’re hosting a webinar, live demo, conference session, or virtual event.
- Multiple people are speaking and coordination risk is high.
- You need promotion + follow-up sequencing tied to pipeline/activation.

Start 2–4 weeks ahead (or faster for small events).

## INPUTS (MINIMUM)
- Event type, date/time, target audience
- Desired outcome (registrations, demos, pipeline, activation)
- Speakers + roles

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Existing content to reuse (slides, demos)
- Promotion channels and budget
- Tech platform details (Zoom, Hopin, Riverside, etc.)

## PROCESS
1. **Define the event promise**: who it’s for + what they’ll be able to do after.
2. **Write the run-of-show**: minute-by-minute agenda with speaker handoffs.
3. **Assign roles**: host, speaker, moderator, producer, chat/Q&A owner, tech backup.
4. **Build the promotion schedule**: email, social, partners, paid (if used).
5. **Create scripts**: intro, transitions, CTA moments, Q&A prompts, closing.
6. **Tech checks**: rehearsal checklist, screen share, audio, backup recording.
7. **Post-event follow-up**: replay email, key takeaways, CTA, sales handoff rules.
8. **Retro**: what improved conversion, what to change next time.

## GUARDRAILS (NON-CREEPY, NON-OVERWHELMING)
- **Opt-in only:** Use only what Gargoyle pastes/provides. Never imply you “saw” private data.
- **Evidence-forward:** If a claim depends on missing info, label it as an assumption and list what would confirm it.
- **Decision-first:** Don’t dump frameworks. Produce an artifact Gargoyle can use immediately.
- **Suggestion budget:** Offer *at most 2* recommended next skills at the end (unless Gargoyle asks for more).
- **No creep:** No sentiment guesses about people. No “what they really mean.” Stick to facts, incentives, commitments, and risks.
- **Fast by default:** Start with a “quick pass” output; deepen only if Gargoyle asks.
- **No over-production:** If goal is pipeline, prioritize clarity of offer and follow-up over fancy visuals.
