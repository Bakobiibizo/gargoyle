# Content: Copywriting (Longform)

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Produce longform content that persuades skeptical readers. Emphasis on clear thesis, mechanisms, proof, and a single CTA — not filler.

## WHEN TO USE
Kick this off when:
- Writing a blog post, guide, manifesto, or deep landing page.
- You need a thought leadership piece tied to pipeline or category narrative.
- You want to convert customer pain into a compelling asset.

Works best with a clear ICP and proof points.

## INPUTS (MINIMUM)
- Topic + target audience
- One-sentence thesis (what you want the reader to believe)
- CTA (what you want them to do)

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Proof points: data, customer quotes, internal metrics, examples
- SEO keyword (if applicable)
- Competitive content you want to beat

## PROCESS
1. **Outline first**: H1/H2s, each section’s claim, and required proof per claim.
2. **Collect proof**: add specific examples, numbers, and counterpoints.
3. **Draft fast**: short paragraphs, concrete language, no filler.
4. **Skeptic edit**: remove claims without evidence; tighten mechanisms.
5. **Structure edit**: improve flow, add signposts, ensure CTA is earned.
6. **Final polish**: clarity, tone, grammar, links, formatting.
7. **Distribution hooks**: generate 5–10 shortform derivatives and a newsletter version.

## OUTPUT FORMAT
### LONGFORM OUTLINE (proof-aware)
- H1:
- Hook:
- Section 1 claim + proof:
- Section 2 claim + proof:
- Section 3 claim + proof:
- Objections + counters:
- CTA:

### DRAFT (publish-ready)
- Title options (5)
- Final article
- Meta description (if needed)
- CTA block

### DERIVATIVES
- 3 LinkedIn posts
- 5 social snippets
- 1 newsletter section

## GUARDRAILS (NON-CREEPY, NON-OVERWHELMING)
- **Opt-in only:** Use only what Gargoyle pastes/provides. Never imply you “saw” private data.
- **Evidence-forward:** If a claim depends on missing info, label it as an assumption and list what would confirm it.
- **Decision-first:** Don’t dump frameworks. Produce an artifact Gargoyle can use immediately.
- **Suggestion budget:** Offer *at most 2* recommended next skills at the end (unless Gargoyle asks for more).
- **No creep:** No sentiment guesses about people. No “what they really mean.” Stick to facts, incentives, commitments, and risks.
- **Fast by default:** Start with a “quick pass” output; deepen only if Gargoyle asks.
- **No content inflation:** If you can say it in 800 words, don’t write 2,000.
