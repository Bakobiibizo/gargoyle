# Organizing: Meeting Cadence + Rituals

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Replace reactive meetings with a small set of high-leverage rituals that produce artifacts: plans, decisions, and commitments. Designed to protect Gargoyle’s time and reduce coordination drag.

## WHEN TO USE
Kick this off when:
- Meetings are frequent but decisions are rare.
- Work is slipping because nobody has a predictable review cadence.
- You’re scaling the team or adding agencies.
- Launch/campaign load is increasing.

Revisit when headcount doubles or the channel mix changes.

## INPUTS (MINIMUM)
- Team structure (roles) and key stakeholders (Sales, Product, CS)
- Current pain points (too many meetings, lack of alignment, slow approvals)
- Top outcomes for the next 30–90 days

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Current calendar snapshot (optional)
- Existing meeting list (titles + frequency)
- Preferences: async-first vs synchronous-heavy

## PROCESS
1. **Define the minimum ritual set** (defaults below).
2. **Set agendas that force decisions** (every meeting ends with owners + dates).
3. **Define pre-read requirements**: what must be sent 24h prior (or meeting auto-cancels).
4. **Set review gates**: creative review, launch readiness, performance review.
5. **Create templates** for each meeting output (notes → actions).
6. **Add escalation path**: what issues go to Gargoyle vs handled by DRIs.
7. **Audit and delete meetings** that don’t produce decisions or shipped output.

## OUTPUT FORMAT
### RECOMMENDED CADENCE (default)

**Weekly Performance Review (60 min)**
- Output: Weekly Performance Pack + decisions + experiment list

**Campaign/Launch Standup (15 min, during active pushes)**
- Output: blockers + owner + unblock plan

**Creative Review (30 min, 1–2x/week)**
- Output: approve/iterate/kill decisions; next delivery date

**Monthly Planning (90 min)**
- Output: priorities, budget shifts, channel bets, resourcing

**Quarterly Strategy Reset (2–3h)**
- Output: ICP/messaging refresh, big bets, calendar, targets

### AGENDA TEMPLATE (decision-first)
1) Scoreboard (5 min)
2) What changed (10 min)
3) Decisions (20 min)
4) Experiments + ship list (20 min)
5) Owners + due dates (5 min)

### PRE-READ RULE
- If pre-read not sent 24h before, meeting auto-reschedules or converts to async.

## GUARDRAILS (NON-CREEPY, NON-OVERWHELMING)
- **Opt-in only:** Use only what Gargoyle pastes/provides. Never imply you “saw” private data.
- **Evidence-forward:** If a claim depends on missing info, label it as an assumption and list what would confirm it.
- **Decision-first:** Don’t dump frameworks. Produce an artifact Gargoyle can use immediately.
- **Suggestion budget:** Offer *at most 2* recommended next skills at the end (unless Gargoyle asks for more).
- **No creep:** No sentiment guesses about people. No “what they really mean.” Stick to facts, incentives, commitments, and risks.
- **Fast by default:** Start with a “quick pass” output; deepen only if Gargoyle asks.
- **No meeting theater:** If there’s no decision or shipped output, cancel it.
