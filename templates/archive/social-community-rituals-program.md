# Community: Rituals + Programming

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Build community engagement through predictable rituals that provide value: prompts, AMAs, challenges, office hours, showcases — with clear goals and measurement.

## WHEN TO USE
Kick this off when:
- You’re launching a community (Slack/Discord/forum).
- Engagement is low and feels like a ghost town.
- You want community to drive retention, advocacy, or support deflection.

Start with a 30-day program and iterate.

## INPUTS (MINIMUM)
- Community purpose (support, learning, advocacy, networking)
- Target member profile (who should join)
- Platform (Slack, Discord, Circle, etc.)

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Current engagement metrics (members, active, posts)
- Moderation resources and constraints
- Content assets you can use as prompts

## PROCESS
1. **Define value promise**: why members show up and what success looks like.
2. **Design onboarding**: welcome message, rules, first actions, intro prompt.
3. **Create ritual set**: weekly prompt, monthly AMA, office hours, showcases, recognition.
4. **Programming calendar (30 days)**: what happens each week.
5. **Recognition system**: highlight wins, member spotlights, badges (lightweight).
6. **Feedback loop**: collect themes and route insights to product/marketing.
7. **Measurement**: active rate, posts/member, retention, referrals (as applicable).
