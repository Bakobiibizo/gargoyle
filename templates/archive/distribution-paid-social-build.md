# Distribution: Paid Social Build

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Design paid social that can be iterated: audience hypotheses, creative system, offer framing, testing plan, and measurement — not one-off ads.

## WHEN TO USE
Kick this off when:
- Launching a new paid social campaign.
- Creative fatigue is reducing performance.
- You want to scale spend with a disciplined testing system.

Pair with `content-ad-creative-concepts` and `staging-paid-campaign-preflight`.

## INPUTS (MINIMUM)
- Platform(s) and target audience
- Offer/CTA
- Landing page destination

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Past performance by audience/creative
- Budget and CPA targets
- Creative constraints (video vs static)

## PROCESS
1. **Audience hypotheses**: interest, lookalike, retargeting, list-based (if available).
2. **Creative system**: concepts, angles, formats, proof points.
3. **Campaign structure**: separate prospecting vs retargeting; clear naming.
4. **Testing plan**: isolate variables (creative first, then audience/bid).
5. **Placement strategy**: where to start vs expand.
6. **Measurement spec**: UTMs, pixel events, conversion definitions, attribution windows.
7. **Iteration cadence**: creative refresh schedule and weekly decision rules.

## OUTPUT FORMAT
### PAID SOCIAL BUILD PLAN
| Campaign | Audience | Objective | Creative concepts | Landing page | KPI | Budget |
|---|---|---|---|---|---|---|

### TESTING SEQUENCE
1) Test creative concepts (constant audience)
2) Scale winning concepts
3) Test audience expansions
4) Test landing page variants

### CREATIVE REFRESH RULE
- Refresh creatives every __ days OR when frequency > __ OR CTR drops __%

## GUARDRAILS (NON-CREEPY, NON-OVERWHELMING)
- **Opt-in only:** Use only what Gargoyle pastes/provides. Never imply you “saw” private data.
- **Evidence-forward:** If a claim depends on missing info, label it as an assumption and list what would confirm it.
- **Decision-first:** Don’t dump frameworks. Produce an artifact Gargoyle can use immediately.
- **Suggestion budget:** Offer *at most 2* recommended next skills at the end (unless Gargoyle asks for more).
- **No creep:** No sentiment guesses about people. No “what they really mean.” Stick to facts, incentives, commitments, and risks.
- **Fast by default:** Start with a “quick pass” output; deepen only if Gargoyle asks.
- **Don’t blame targeting for bad creative:** Creative usually fails first.
