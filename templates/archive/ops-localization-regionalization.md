# Ops: Localization + Regionalization

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Scale marketing across regions without breaking brand or wasting effort. Output defines what gets localized, how, who owns it, and how success is measured per region.

## WHEN TO USE
Kick this off when:
- Expanding into new geographies/languages.
- Regional teams request localized assets.
- Performance differs by region and you need structured adaptation.

Start with a pilot region to validate workflow.

## INPUTS (MINIMUM)
- Target regions/languages
- Core assets to localize (LPs, emails, ads, decks)
- Ownership (central vs regional teams)

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Regional constraints (legal, cultural, channel availability)
- Translation resources (vendors, in-house)
- Region-specific ICP differences

## PROCESS
1. **Prioritize regions**: revenue potential × readiness × channel access.
2. **Define localization scope**: translate vs transcreate; what must remain consistent.
3. **Set workflow**: source copy → translation → regional review → QA → publish.
4. **Define glossary and style**: key terms, brand voice in region.
5. **Channel adaptation**: what works per region (platform differences).
6. **Measurement**: region scorecards and feedback loops.
7. **Governance**: version control and update propagation.

## GUARDRAILS (NON-CREEPY, NON-OVERWHELMING)
- **Opt-in only:** Use only what Gargoyle pastes/provides. Never imply you “saw” private data.
- **Evidence-forward:** If a claim depends on missing info, label it as an assumption and list what would confirm it.
- **Decision-first:** Don’t dump frameworks. Produce an artifact Gargoyle can use immediately.
- **Suggestion budget:** Offer *at most 2* recommended next skills at the end (unless Gargoyle asks for more).
- **No creep:** No sentiment guesses about people. No “what they really mean.” Stick to facts, incentives, commitments, and risks.
- **Fast by default:** Start with a “quick pass” output; deepen only if Gargoyle asks.
- **Don’t translate bad copy:** Fix core messaging first, then localize.
