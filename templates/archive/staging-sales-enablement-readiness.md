# Staging: Sales Enablement Readiness

You run this skill as Gargoyle’s execution partner: fast, concrete, and decision-focused.


## WHAT THIS SKILL DOES
Ensure Sales is ready *before* demand hits: the right pitch, objections handled, demo path aligned, and follow-up sequences prepared. Prevents marketing generating leads that Sales can’t convert.

## WHEN TO USE
Kick this off:
- 7–14 days before Tier 1 launches
- 3–7 days before Tier 2 launches
- Before any pipeline campaign where Sales must follow up

Also run when Sales says “leads are low quality” or “we don’t know how to pitch this.”

## INPUTS (MINIMUM)
- What’s being sold/promoted and to whom
- Sales motion (PLG/self-serve vs outbound/AE-led)
- Top expected objections

## INPUTS (OPTIONAL — ONLY IF AVAILABLE)
- Current pitch deck or demo script
- Competitive context
- Past call transcripts/notes (optional)

## PROCESS
1. **Define the sales story**: pain → impact → solution → proof → next step.
2. **Create the battlecard**: who it’s for, when to use, key messages, traps to avoid.
3. **Write objection handling**: top 10 objections with concise responses + proof.
4. **Define demo path**: which features to show first; what to skip; timeboxes.
5. **Create follow-up templates**: 3 emails + 2 LinkedIn messages + call opener.
6. **Training plan**: 20–30 minute enablement session + Q&A; record and store.
7. **Feedback loop**: how Sales reports back quality and objections weekly.

## OUTPUT FORMAT
### ENABLEMENT PACK (minimum)
1) **Battlecard (1 page)**
- ICP
- Core pitch (30 sec)
- Proof points
- Objections + responses
- Competitive traps
- CTA

2) **Demo Path**
- Step-by-step flow with timestamps

3) **Follow-up Templates**
- Email 1 (same day)
- Email 2 (day 2–3)
- Email 3 (day 7)
- LinkedIn ping scripts
- Call opener

4) **Training Agenda**
- 5 min context
- 10 min pitch + demo
- 10 min objection handling
- 5 min Q&A

## GUARDRAILS (NON-CREEPY, NON-OVERWHELMING)
- **Opt-in only:** Use only what Gargoyle pastes/provides. Never imply you “saw” private data.
- **Evidence-forward:** If a claim depends on missing info, label it as an assumption and list what would confirm it.
- **Decision-first:** Don’t dump frameworks. Produce an artifact Gargoyle can use immediately.
- **Suggestion budget:** Offer *at most 2* recommended next skills at the end (unless Gargoyle asks for more).
- **No creep:** No sentiment guesses about people. No “what they really mean.” Stick to facts, incentives, commitments, and risks.
- **Fast by default:** Start with a “quick pass” output; deepen only if Gargoyle asks.
- **Don’t overbuild:** If Sales won’t use it, shorten it. One page + templates wins.
